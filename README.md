# Supplementary Research Artifacts for ADD+k Thesis Evaluation

This repository contains supplementary research artifacts for the thesis:

**Enhancing Architectural Decision Traceability in Attribute-Driven Design through Residuality Matrix Analysis**

Author: **Aniia Kotomceva**  
Institution: **Innopolis University**  
Year: **2026**

## Repository Purpose

The repository provides anonymized empirical and analytical materials used in the evaluation of the ADD+k method. ADD+k is an extension of Attribute-Driven Design (ADD) that introduces residuality matrix analysis into the architectural design process.

The uploaded materials support the evaluation of the following research questions:

- **RQ1:** Does ADD+k improve the traceability of architectural decisions compared to basic Attribute-Driven Design?
- **RQ2:** Does ADD+k improve the quality and consistency of architectural decision justification compared to basic ADD?
- **RQ3:** Does ADD+k increase the comparability of architectural solutions between groups without imposing excessive effort?

## Contents of the Repository

### 1. `ADD_student_artifacts_anonymized.xlsx`

This file contains anonymized student artifacts from the control group that used the basic ADD method.

The file includes student-produced architectural design evidence across the ADD iterations. The artifacts were anonymized before publication: personal identifiers such as student names and email addresses were removed or replaced with generated participant identifiers.

This file is mainly used as empirical input for comparing the basic ADD group against the ADD+k group.

### 2. `ADD_K_student_artifacts_anonymized.xlsx`

This file contains anonymized student artifacts from the experimental group that used the ADD+k method.

The file includes student-produced architectural design evidence across the ADD+k iterations, including the use of residuality-matrix-based reasoning where applicable. Personal identifiers were removed or replaced with generated participant identifiers.

This file is used to evaluate whether the additional residuality matrix mechanism influenced traceability, justification quality, consistency, and comparability of architectural reasoning.

### 3. `ADD_table.xlsx`

This file contains the structured analytical table for the ADD group.

It is used to convert qualitative student submissions into comparable evaluation data. The table supports the analysis of architectural coverage, traceability indicators, justification quality, and comparability-related metrics for the control group.

### 4. `From_Student_Submissions_to_Binary_Evidence_Tables.xlsx`

This file documents the transformation of qualitative student submissions into binary evidence tables.

The purpose of this artifact is to make the evaluation process more transparent and reproducible. Since student submissions were written in free form and differed in structure, depth, and terminology, the qualitative evidence was converted into binary indicators.

The binary coding principle was:

- `1` — the indicator is explicitly present and supported by the submitted artifact;
- `0` — the indicator is absent, implicit, contradictory, weakly justified, or not reliably confirmable.

The file includes three analytical dimensions:

1. **Quality Attribute Coverage**  
   Captures whether the architecture explicitly supports relevant quality attributes and subcharacteristics.

2. **Architectural Solution Categories**  
   Captures whether the submission includes meaningful architectural mechanisms expressed through generalized architectural categories rather than exact technologies.

3. **Research-Oriented Evaluation Metrics**  
   Captures traceability, justification quality, internal consistency, comparability, and effort-related indicators.

This file is central for explaining how the empirical evidence was converted into comparable data for answering RQ1, RQ2, and RQ3.

### 5. `Evaluation_and_Discussion.pptx`

This presentation summarizes the evaluation and discussion results of the empirical study.

It includes the main interpretation of the comparison between ADD and ADD+k, including observations about traceability, justification quality, comparability, variance between student submissions, and limitations of the study.

This file can be used as supplementary presentation material for the thesis discussion and defense.

### 6. `Literature_Review.xlsx`

This file contains the literature review register used during the preparation of the thesis.

It documents reviewed sources related to Attribute-Driven Design, architectural decision traceability, residuality theory, architecture evaluation methods, and empirical studies of software architecture design methods.

The file supports the theoretical and methodological foundation of the thesis.

## Data Anonymization

All uploaded empirical artifacts were prepared for public sharing. Personal identifiers were removed before publication. Student names, email addresses, and other direct identifiers are not included in the published research dataset.

The repository is intended to provide transparency and reproducibility while preserving participant privacy.

## How These Artifacts Support the Thesis

The artifacts support the thesis evaluation as follows:

- The anonymized student artifacts provide the empirical basis for comparing ADD and ADD+k.
- The binary evidence tables explain how qualitative submissions were transformed into measurable indicators.
- The analytical tables support the calculation of coverage, traceability, justification, comparability, and effort-related metrics.
- The presentation summarizes the main evaluation findings and discussion points.
- The literature review register documents the theoretical sources used to frame the research problem and methodology.
